Executive Summary

Because Netlify Deploy Previews built from an older commit lacking the new server-side formatting functions, certifications and licensed ingredients rendered as plain text on preview builds, while local builds included the correct box HTML and styles.

Root Cause Analysis

Suspected root cause:

  • Primary Hypothesis: Deploy preview used a stale branch/commit missing formatCertifications() and formatLicensedIngredients() in src/pages/brands/[slug].astro.
  • Contributing Factors: Netlify cache and/or PR source targeting the wrong commit; CSS wasn’t globally present previously (now added to src/styles/base.css).

Evidence

Code Reference

Functions present in src/pages/brands/[slug].astro:

formatCertifications 
function formatCertifications(certifications) {
  if (!certifications || !Array.isArray(certifications) || certifications.length === 0) return '—';
  const boxes = certifications.map(cert => `${cert}`).join(' ');
  return `
${boxes}
`;
}
formatLicensedIngredients 
function formatLicensedIngredients(ingredients) {
  if (!ingredients || typeof ingredients !== 'string' || ingredients.trim() === '') return '—';
  const list = ingredients.split(',').map(s => s.trim()).filter(Boolean);
  const boxes = list.map(x => `${x}`).join(' ');
  return `
${boxes}
`;
}

CSS Inclusion

Global styles for boxes added to src/styles/base.css to guarantee inclusion in builds.

Proposed Fix

  1. Immediate Mitigation: Ensure SSR formatting functions and box CSS are in the feature branch (verified; CSS added globally).
    Validation: Local build shows correct box HTML; functions exist server-side.
  2. Short-term Fix: Open a PR from feature/database-update-licensed-ingredients to main; clear Netlify cache and trigger a fresh deploy preview from the PR.
    Validation: Preview page HTML contains cert-box and ingredient-box spans.
  3. Long-term Solution: Disable Netlify cached deploys for schema/code changes or add “Clear cache and deploy” to release checklist; add an HTML snapshot test for brand page boxes.
    Validation: Future previews consistently render boxes without manual cache clears.
